# Вклад в проект

Спасибо за интерес к проекту! Мы рады любому вкладу.

## Как внести вклад

### Сообщения об ошибках

Если вы нашли ошибку, пожалуйста:

1. Проверьте, что ошибка еще не была сообщена в Issues
2. Создайте новый Issue с детальным описанием:
   - Шаги для воспроизведения
   - Ожидаемое поведение
   - Фактическое поведение
   - Версия Go и ОС

### Предложения улучшений

Для предложений новых функций:

1. Откройте Issue с описанием функции
2. Объясните, почему эта функция будет полезна
3. Приведите примеры использования

### Pull Requests

1. Форкните репозиторий
2. Создайте ветку для вашей функции (`git checkout -b feature/amazing-feature`)
3. Сделайте коммит изменений (`git commit -m 'Add amazing feature'`)
4. Запушьте в ветку (`git push origin feature/amazing-feature`)
5. Откройте Pull Request

### Стандарты кода

- Следуйте стандартному стилю Go (используйте `go fmt`)
- Добавляйте тесты для новых функций
- Обновляйте документацию при необходимости
- Убедитесь, что все тесты проходят (`make test`)

### Структура коммитов

Используйте понятные сообщения коммитов:

- `feat:` для новых функций
- `fix:` для исправлений ошибок
- `docs:` для изменений в документации
- `test:` для добавления тестов
- `refactor:` для рефакторинга

Пример: `feat: add support for custom fonts injection`

## Тестирование

Перед отправкой PR убедитесь, что:

```bash
make test    # Все тесты проходят
make lint    # Нет ошибок линтера (если установлен golangci-lint)
make format  # Код отформатирован
```

## Лицензия

Внося вклад в проект, вы соглашаетесь, что ваш код будет лицензирован под MIT License.
